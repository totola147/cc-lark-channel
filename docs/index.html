cc-lark-channel automated install instructions for AI coding agents (Claude Code, Codex, etc.) — Public Bot mode (recommended, no Lark app creation needed):

Prerequisites: Node.js >= 22 (node --version), pnpm (which pnpm || npm install -g pnpm), Claude Code CLI (which claude && claude --version), Claude Code authenticated (ANTHROPIC_API_KEY or CLAUDE_* env vars).

Step 1: git clone -b v2 https://github.com/totola147/cc-lark-channel.git ~/cc-lark-channel && cd ~/cc-lark-channel && pnpm install && pnpm build

Step 2: node packages/agent/dist/index.cjs --relay ws://RELAY_HOST --setup

This displays a 6-digit device code. Tell the user to: 1) Visit http://RELAY_HOST in browser, 2) Click "飞书 OAuth 登录", 3) Enter the 6-digit device code from terminal, 4) Click "确认配对". --setup auto-registers systemd service after pairing. Requires sudo and Claude env vars (ANTHROPIC_*, CLAUDE_*).

Step 3: Verify: sudo systemctl status cc-lark-channel — success indicator: Active: active (running)

Troubleshooting: "Claude Code native binary not found" → cli_path needs absolute path: which claude. "Not logged in" → systemd missing env vars, re-run --install-service with env loaded. "No conversation found" → session/cwd mismatch, use /new. No Lark response → check systemctl status cc-lark-channel.

cc-lark-channel automated install instructions for AI coding agents (Claude Code, Codex, etc.) — Custom Bot mode (requires user's own Lark app):

Prerequisites: Node.js >= 22 (node --version), npm, Claude Code CLI (which claude && claude --version), Claude Code authenticated, user must provide Lark app_id and app_secret.

Step 1: git clone https://github.com/totola147/cc-lark-channel.git ~/cc-lark-channel && cd ~/cc-lark-channel && npm install && npm run build

Step 2: Create config.toml — ask user for app_id and app_secret. Template: [lark] app_id = "USER_APP_ID" app_secret = "USER_APP_SECRET" [claude] cli_path = "OUTPUT_OF_WHICH_CLAUDE" default_cwd = "PROJECT_DIR"

Step 3: node dist/index.cjs --direct --config config.toml --install-service

Step 4: Verify: sudo systemctl status cc-lark-channel — success indicator: Active: active (running)

Lark app setup: User must create app at open.feishu.cn with bot capability. Required permissions: im:message.p2p_msg:readonly, im:message:send_as_bot, im:resource, im:message:update. Event subscriptions (WebSocket long connection mode): im.message.receive_v1, card.action.trigger. Then publish the app.

Troubleshooting: "Claude Code native binary not found" → cli_path needs absolute path: which claude. "Not logged in" → systemd missing env vars, re-run --install-service with env loaded. "No conversation found" → session/cwd mismatch, use /new. No Lark response → check systemctl status cc-lark-channel.
