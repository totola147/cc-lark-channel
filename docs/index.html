cc-lark-channel

通过飞书操控 Claude Code，随时随地推进项目开发

公共 Bot（推荐）
自有 Bot
1
安装并启动

在你的 Claude Code 所在机器上执行（需要 Node.js ≥ 22 + pnpm）：

git clone -b v2 https://github.com/totola147/cc-lark-channel.git
cd cc-lark-channel && pnpm install && pnpm build
node packages/agent/dist/index.cjs --relay ws://43.153.201.61:9000 --setup

终端会显示 6 位设备码，继续下一步。

2
飞书认证 + 配对

点击下方按钮完成飞书 OAuth 登录，然后输入设备码：

飞书 OAuth 登录
✓
完成

服务已注册并启动。现在可以在飞书中给 Bot 发消息，Claude Code 会响应。

常用命令：/cd /attach /bg /fg /sessions /help

1
创建飞书应用

在 飞书开放平台 创建企业自建应用：

  • 启用「机器人」能力
  • 权限：im:message.p2p_msg:readonly、im:message:send_as_bot、im:resource、im:message:update
  • 事件订阅（长连接模式）：im.message.receive_v1、card.action.trigger
  • 发布应用
2
安装并配置

在你的 Claude Code 所在机器上执行：

git clone https://github.com/totola147/cc-lark-channel.git
cd cc-lark-channel && npm install && npm run build

# 配置
cp config.example.toml config.toml
# 编辑 config.toml 填入 app_id 和 app_secret
# 设置 claude.cli_path 为 claude 的绝对路径

# 启动
node dist/index.cjs
3
注册系统服务（可选）
node dist/index.cjs --install-service

开机自启、崩溃自动重启。

For AI Agents

以下是结构化安装指令，供 AI coding agent（Claude Code、Codex 等）自动执行。

# ============================================================
# cc-lark-channel 自动化安装指令（供 AI Agent 执行）
# ============================================================
#
# 前置条件检查：
#   - Node.js >= 22: node --version
#   - pnpm: which pnpm || npm install -g pnpm
#   - Claude Code CLI: which claude && claude --version
#   - Claude Code 已认证: 确认 ANTHROPIC_API_KEY 或 CLAUDE_* 环境变量存在
#
# ============================================================
# 模式 A：公共 Bot（推荐，无需创建飞书应用）
# ============================================================
#
# Step 1: 克隆并构建
git clone -b v2 https://github.com/totola147/cc-lark-channel.git ~/cc-lark-channel
cd ~/cc-lark-channel && pnpm install && pnpm build
#
# Step 2: 启动配对（交互式，需要用户操作）
node packages/agent/dist/index.cjs --relay ws://43.153.201.61:9000 --setup
#
# 执行后终端显示 6 位设备码。
# 告诉用户：
#   1. 浏览器访问 http://43.153.201.61:9000
#   2. 点击「飞书 OAuth 登录」
#   3. 输入终端显示的 6 位设备码
#   4. 点击「确认配对」
#
# --setup 会在配对成功后自动注册 systemd 服务。
# 需要 sudo 权限和 Claude 相关环境变量（ANTHROPIC_*、CLAUDE_*）。
#
# Step 3: 验证
sudo systemctl status cc-lark-channel
# 成功标志: Active: active (running)
#
# ============================================================
# 模式 B：自有 Bot（需要用户提供飞书 app_id/app_secret）
# ============================================================
#
# Step 1: 克隆并构建
git clone https://github.com/totola147/cc-lark-channel.git ~/cc-lark-channel
cd ~/cc-lark-channel && npm install && npm run build
#
# Step 2: 创建配置（向用户询问 app_id 和 app_secret）
cat > config.toml << 'EOF'
[lark]
app_id = "ASK_USER_FOR_APP_ID"
app_secret = "ASK_USER_FOR_APP_SECRET"

[claude]
cli_path = "REPLACE_WITH_OUTPUT_OF_WHICH_CLAUDE"
default_cwd = "REPLACE_WITH_PROJECT_DIR"
EOF
#
# Step 3: 注册服务
node dist/index.cjs --direct --config config.toml --install-service
#
# Step 4: 验证
sudo systemctl status cc-lark-channel
#
# ============================================================
# 故障排查
# ============================================================
# "Claude Code native binary not found" → cli_path 需要绝对路径: which claude
# "Not logged in" → systemd 缺少环境变量，重新执行 --install-service（确保 env 已加载）
# "No conversation found" → session 与 cwd 不匹配，用 /new 开始新会话
# 飞书无响应 → 检查 systemctl status cc-lark-channel 是否 active
# ============================================================
