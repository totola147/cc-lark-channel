cc-lark-channel 使用文档

通过飞书远程操控 Claude Code，随时随地推进项目开发

特性

  • 双模式接入 — 公共 Bot（零配置扫码）或自有 Bot（企业私有化）
  • OAuth + 设备码配对 — 安全便捷的接入流程
  • 交互式审批卡片 — 工具调用时发送按钮卡片，手机点一下即可审批
  • 流式状态卡片 — 实时展示 thinking、工具调用、输出，原地更新不刷屏
  • 后台会话 — 长任务放后台继续跑，切到新会话继续工作
  • 工作区群组 — 每个项目一个飞书群，独立会话池
  • 消息队列 — 生成中发送的消息自动排队，空闲后依次执行
  • 中断机制 — ! 前缀中断当前任务，/stop 停止生成
  • 会话接管 — /attach 接管已有 CLI 会话
  • 图片支持 — 发截图给 Claude 分析
  • 零公网 IP — 用户侧无需开放端口
  • 系统服务 — 一键注册 systemd，开机自启

设计理念

cc-lark-channel 提供两种开发模式，对应不同的工作场景：

Spec 模式 — 工作区群组（群聊）

每个项目一个飞书群，适合结构化、目标明确的开发任务。

  • 一个群 = 一个项目，天然隔离上下文
  • 独立的会话池：前台会话处理当前任务，后台会话并行跑长任务
  • 群名自动从项目元数据（package.json 等）读取，一目了然
  • 适合：功能开发、bug 修复、代码审查、持续迭代
  • 思路：像写 spec 一样明确目标，Claude 按计划执行

Vibe 模式 — 私聊

直接在私聊中与 Claude Code 对话，适合自由探索、快速实验。

  • 无需预设项目，随时切换目录（/cd）
  • 快速原型、一次性脚本、跨项目操作
  • 想到什么做什么，不需要提前规划
  • 适合：探索想法、快速验证、临时任务、学习实验
  • 思路：像 vibe coding 一样跟着感觉走，Claude 即时响应

两种模式互补：用私聊快速验证想法，确定方向后创建工作区群组持续推进。

架构

公共 Bot 模式: 飞书用户 → 公共 Bot → Cloud Relay (WebSocket) → 本地 Agent → Claude Code 自有 Bot 模式: 飞书用户 → 自有 Bot → 本地 Agent → Claude Code（直连）

快速开始

方式一：公共 Bot推荐

在 Claude Code 所在机器上执行：

# 1. 克隆并构建（需要 Node.js >= 22 + pnpm）
git clone -b v2 https://github.com/totola147/cc-lark-channel.git
cd cc-lark-channel && pnpm install && pnpm build

# 2. 首次启动（交互式配对）
node packages/agent/dist/index.cjs --setup

配对流程：

  1. 终端显示 6 位设备码
  2. 访问 Landing Page，点击「飞书 OAuth 登录」
  3. 输入设备码，点击「确认配对」
  4. 自动注册 systemd 服务，开机自启

方式二：自有 Bot

适合企业/隐私需求，需要自己创建飞书应用。

飞书应用配置：

  1. 在 飞书开放平台 创建企业自建应用
  2. 启用「机器人」能力
  3. 添加权限：im:message.p2p_msg:readonly、im:message:send_as_bot、im:resource、im:message:update
  4. 事件订阅（长连接模式）：im.message.receive_v1、card.action.trigger
  5. 发布应用
# 克隆并构建
git clone -b v2 https://github.com/totola147/cc-lark-channel.git
cd cc-lark-channel && pnpm install && pnpm build

# 配置
cp config.example.toml config.toml
# 编辑 config.toml 填入 app_id / app_secret

# 启动
node packages/agent/dist/index.cjs --direct --config config.toml

# 注册系统服务（可选）
node packages/agent/dist/index.cjs --install-service

命令列表

命令说明
/new开始新会话
/stop停止当前生成
/status查看会话状态（模型、token、队列等）
/sessions列出所有会话（前台/后台）
/bg [name]将当前会话移到后台继续执行
/fg <id>将后台会话切到前台
/kill <id>终止后台会话
/kill all终止所有会话
/attach <id>接管已有 CLI 会话
/cd <path>切换工作目录（启动新会话）
/workspace <path>为项目创建飞书群组工作区
/workspaces列出所有工作区
/close关闭当前工作区群组
/mode <mode>切换权限模式（default / acceptEdits / bypassPermissions）
/model <name>切换模型
/update拉取最新代码并重启
/help显示帮助

特殊输入

输入效果
! <text>中断当前生成，立即执行新指令
普通文本发送给 Claude（忙时自动排队）
图片下载后以 base64 发送给 Claude 分析

工作区群组

每个项目可以创建独立的飞书群作为工作区，群内拥有独立的会话池：

  • 私聊中发送 /workspace /path/to/project 创建群组
  • 群名自动从 package.json / pyproject.toml 等读取项目名
  • 自动 attach 到该路径下最近的 CLI session
  • 每个用户最多 20 个工作区
  • 群内发送 /close 解散群组

权限模式

模式行为
default所有工具调用需要审批（发送卡片）
acceptEdits文件编辑自动通过，其他需审批
bypassPermissions所有操作自动通过（谨慎使用）

自动更新

Agent 启动时自动检查更新（git pull + rebuild）。也可以随时在飞书中发送 /update 手动触发。

故障排查

问题解决方案
Claude Code native binary not foundcli_path 需要绝对路径，执行 which claude 获取
Not logged insystemd 缺少环境变量，重新执行 --install-service（确保 env 已加载）
No conversation foundsession 与 cwd 不匹配，用 /new 开始新会话
飞书无响应检查 systemctl status cc-lark-channel 是否 active
设备码过期重启 agent 获取新设备码
